控制器通过xib文件创建View

发表于 2017-07-29 | 分类于 Swift笔记

官方文档

If you use a nib file to store your view controller’s view, it is recommended that you specify that nib file explicitly when initializing your view controller. However, if you do not specify a nib name, and do not override the loadView() method in your custom subclass, the view controller searches for a nib file using other means. Specifically, it looks for a nib file with an appropriate name (without the .nib extension) and loads that nib file whenever its view is requested. Specifically, it looks (in order) for a nib file with one of the following names:

  1. If the view controller class name ends with the word ‘Controller’, as in MyViewController , it looks for a nib file whose name matches the class name without the word ‘Controller’, as in MyView.nib .

  2. It looks for a nib file whose name matches the name of the view controller class. For example, if the class name is MyViewController , it looks for a MyViewController.nib file.

翻译

如果使用nib文件存储视图控制器的视图，建议您在初始化视图控制器时明确指定该nib文件。 但是，如果不指定nib名称，并且不要在自定义子类中覆盖loadView（）方法，则视图控制器将使用其他方法搜索nib文件。 具体来说，它查找一个具有适当名称（不带.nib扩展名）的nib文件，并在请求视图时加载该nib文件。 具体来说，对于具有以下名称之一的nib文件来说，它的顺序）：

  1. 如果视图控制器类名以“Controller”一词结尾，它会查找不含Controller的nib文件, 例如: MyView.nib

  2. 它查找名称与视图控制器类的名称相匹配的nib文件。 例如，如果类名是MyViewController，它会查找MyViewController.nib文件。

栗子🌰

1. 首先创建一个工程, 删除自带的 mainStoryboard 和 ViewController.swift 文件

2. 然后创建三个文件

  • MyViewController.swift
  • MyViewController.xib
  • MyView.xib

3.在AppDelegate里面

4.配置两个xib文件

  • 我们需要配置的是xib文件里面的 File’s Owner 和 Owner的 view
  • 点击xib文件, 设置File’s Owner 的类 为 MyViewController

  • 点击 File’s Owner 将弹出的 Outlets 里面的 view 属性连接到 storyboard里面的View
  • 两个xib文件同样配置(颜色要区分开来, 容易检查)

5.运行测试

  1. 先按照上面的测试, 你会发现显示的是 MyViewController.xib 文件里面的View
  2. 然后将nibNme设置为 nil, 你会发现显示的是 MyViewController.nib
  3. 如果你直接调用 MyViewController()方法, 不传入任何参数, 还是会显示MyViewController.nib , 因为你什么都不设置的话, 默认调用 nibName: bundle: 方法, 参数传入nil
  4. 注: bundle 传入nil就代表 mainBundle

6.总结

加载MyViewController控制器的View
  • 正如官网上提到的, 如果你没有为 nibName 赋值, 那么系统首先会自动去 默认bundel 中查找和 MyViewController 类名匹配的nib文件, 如果没有的话, 会查找 MyView 名字的nib文件,
  • 亲测过, 并没有像一些视频里面讲的先调用的是MyView.nib, 其次再调用 MyViewController
